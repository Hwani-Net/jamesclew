# Account Rotation Protocol (Always On)

## 목적
Antigravity Pro 계정이 주간 리밋(Weekly Rate Limit)에 도달했을 때,
컨텍스트 유실 없이 다음 계정으로 세션을 이어받는 표준 절차.

## 리밋 도달 감지 조건
다음 중 하나라도 해당되면 즉시 아래 프로토콜을 실행한다:
- 응답 생성 중 "rate limit", "quota exceeded", "weekly limit reached" 메시지 수신
- 응답 속도가 현저히 저하되어 리밋 임박 판단
- 대표님이 "리밋", "계정 바꿔", "다음 계정" 등을 언급

## 로테이션 프로토콜 (순서 엄수)

### Step 1: 즉시 현재 상태 저장
현재 작업 내용을 아래 두 곳에 즉시 기록한다:

```
docs/PROJECT_CONTEXT.md  ← 진행 중인 태스크, 완료/미완료 체크리스트, 결정 사항
```

기록 포맷:
```markdown
## 🔄 세션 핸드오프 (YYYY-MM-DD HH:MM)
- **중단 지점**: [지금 하고 있던 작업 1줄 요약]
- **완료된 것**: [✅ 체크리스트]
- **남은 것**: [⬜ 체크리스트]
- **중요 컨텍스트**: [다음 세션에서 반드시 알아야 할 결정/이슈]
- **다음 액션**: [재개 시 첫 번째로 해야 할 일]
```

### Step 2: 대표님에게 알림
다음 메시지를 정확히 출력한다:

```
⚠️ 주간 리밋 도달. 현재 상태를 PROJECT_CONTEXT.md에 저장했습니다.
새 Antigravity 창에서 아래 명령어를 붙여넣으세요:

/이어서
```

### Step 3: 다음 계정 세션에서의 복구 절차
새 계정 세션 시작 시 `/이어서` 또는 `뇽죵아 이어서 해줘` 를 수신하면:

1. `docs/PROJECT_CONTEXT.md` 읽기 → 핸드오프 섹션 확인
2. `docs/PITFALLS.md` 읽기 → 알려진 함정 재숙지
3. 대표님에게 1줄 현황 보고 후 즉시 작업 재개

보고 포맷:
```
✅ [이전 계정] 세션 복구 완료.
중단 지점: [작업 요약]
지금부터: [다음 액션] 시작합니다.
```

## 예방 규칙
- 대형 작업(PRD 생성, 대규모 리팩터링 등) 시작 전, 현재 잔여 쿼터를 인식하고
  중간 저장 체크포인트를 미리 설정한다.
- 컨텍스트가 65% 이상 차면 `/compact` 실행 전 PROJECT_CONTEXT.md 업데이트 필수.

## 계정 우선순위
계정 번호는 대표님이 관리. 이 규칙은 몇 번 계정이든 동일하게 적용.
