## ADR-001: CLAUDE.md 규칙을 hook으로 구조적 강제 + CLAUDE.md 압축 (2026-04-05)

### 컨텍스트
CLAUDE.md에 규칙을 상세히 적어도 에이전트가 반복 위반함. 4가지 패턴이 각 3-4회 감지됨:
- 선언-미실행 (4회)
- 검수 건너뛰기 (4회)
- 섣부른 결론 (2회)
- 증거 없는 보고 (다수)

Jaroslawicz et al. (2025) 연구: 지시사항 150개 초과 시 준수율 가속 저하.

### 결정
1. 반복 위반 규칙을 Stop/PostToolUse hook으로 이관 (decision:block 방식)
2. CLAUDE.md를 102줄 → 52줄로 압축 (hook 태그 참조 방식)
3. CLAUDE.md에서 규칙을 완전 삭제하지 않고 한 줄 요약 유지 (hook 실패 시 안전망)

### 근거
- 대안 A: CLAUDE.md에 더 강한 어조로 재작성 → 이미 실패한 접근
- 대안 B: hook으로 100% 이관, CLAUDE.md에서 삭제 → hook timeout 시 규칙 완전 소멸 위험
- **선택 C: hook 강제 + CLAUDE.md 한 줄 참조** → 구조적 강제 + 안전망 유지

커뮤니티 합의: "CLAUDE.md는 팀의 집단 기억, hooks는 반사신경"

### 결과
- 새 hook 4건: evidence-first.sh, enforce-review.sh, error-telegram.sh, enforce-execution.sh 패턴 추가
- CLAUDE.md 52줄 (각 규칙에 `[hook 강제: 파일명]` 태그)
- settings.json Stop/PostToolUse에 hook 등록
