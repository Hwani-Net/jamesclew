# /저장 — 세션 저장 + compact

현재 세션의 작업 내용을 저장하고 컨텍스트를 압축합니다.

## 실행 절차 (순서대로 모두 실행)

### 1. 옵시디언 세션 요약 저장
아래 경로에 세션 요약 마크다운 파일을 생성하세요:
`$OBSIDIAN_VAULT/01-jamesclaw/harness/session-{날짜}-{주제}.md`
(환경변수 OBSIDIAN_VAULT 필수. 미설정 시 `~/.harness.env` 확인)

포함 내용:
- 세션 개요 (기간, 커밋 수, 주요 작업)
- 핵심 성과 (변경/추가/수정 목록)
- PITFALLS 신규 항목
- 다음 세션 작업 목록

### 2. 하네스 설계 문서 업데이트
하네스 구조가 변경되었으면:
`$OBSIDIAN_VAULT/01-jamesclaw/harness/harness_design.md`
변경 이력 테이블에 날짜, 변경 내용, 근거 기록.

### 3. git commit + push
미커밋 변경사항이 있으면 커밋. push는 대표님 확인 후.

### 4. compact 자동 실행
위 1-3 완료 후 clipboard 방식으로 `/compact`를 입력합니다 (IME 우회):

```
Step 1: /compact를 클립보드에 복사
  Bash: echo -n "/compact" | clip

Step 2: 포커스 확인 후 붙여넣기 + Enter
  mcp__desktop-control__computer(action: "key", text: "ctrl+v")
  mcp__desktop-control__computer(action: "key", text: "Return")

Step 3: 검증 (필수) — 3초 후 스크린샷으로 compact 실행 여부 확인
  mcp__desktop-control__computer(action: "get_screenshot")
  확인 기준:
  - "Compacted" 또는 "compaction" 텍스트가 보이면 → 성공
  - "> /compact" 가 입력창에 남아 있으면 → Enter 미전달, 재시도
  - 변화 없음 → 포커스 문제, 대표님에게 수동 입력 요청

Step 4: 재시도 (Step 3 실패 시)
  - 입력창 클릭으로 포커스 확보:
    mcp__desktop-control__computer(action: "left_click", coordinate: [입력창 좌표])
  - Step 1~3 반복 (최대 2회)
  - 2회 실패 시 대표님에게 수동 /compact 입력 요청
```

⚠️ desktop-control MCP가 연결되어 있어야 합니다. 연결 안 되어 있으면 대표님에게 수동 입력 요청.
