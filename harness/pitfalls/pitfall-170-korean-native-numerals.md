# pitfall-170 — 한국어 TTS 수사 변환

## 증상
숫자 → 한글 변환을 한자어(일·이·삼·사…)로만 처리하면 단위 명사 앞에서 부자연스러움 발생.
- `2배` → `이배` ❌ (한자어) / `두 배` ✅ (고유 수사)
- `3배` → `삼배` ❌ / `세 배` ✅
- `10명` → `십명` ❌ / `열 명` ✅
- `20명` → `이십명` ❌ / `스무 명` ✅

## 원인
한국어는 수사 체계가 이원적:
1. **고유 수사** (하나·둘·셋…): **고유어 단위 명사** 앞에서 사용
   - 배·명·개·마리·장·권·살·사람·가지·번·차례·채·척·대·그루·잔·송이·시간·달·쌍·켤레…
2. **한자어 수사** (일·이·삼…): **한자어 단위·수량 표현·연도·금액** 앞에서 사용
   - 원·년·월·일·분·초·회·도·% 등

단, **100 이상 + 고유어 단위**는 한자어로 회귀하는 게 자연스러움 (예: `157개` → `백오십칠개`, `370명` → `삼백칠십명`).
**소수점 + 단위**도 한자어 유지 (예: `3.6배` → `삼쩜육배`).

## 해결
`automation/lib/voices_supertonic.py` `kn()` 함수에 2단계 변환 추가:

1. **정수(1~99) + 고유어 단위** → 고유 수사 + 공백 + 단위
   - regex: `(?<![\.\d])(\d+)({고유단위})` (lookbehind 로 소수 형태 제외)
2. **그 외 숫자** → 한자어 (소수점은 `쩜` 사용)

특수 형태:
- `20 + 단위` → "스무" (속성형, "스물" 아님)
- `21 + 단위` → "스물한" (합성형)
- 1·2·3·4 → 한·두·세·네 (단위 앞 attribute form, "하나·둘·셋·넷"이 아님)

## 검증 사례
```
3.6배입니다     → 삼쩜육배입니다  (소수 한자어 유지)
3배에           → 세 배에
2배 차이        → 두 배 차이
10개 항목       → 열 개 항목
20명            → 스무 명
21명            → 스물한 명
99마리          → 아흔아홉 마리
100배           → 백배       (100+ 한자어 회귀)
157개           → 백오십칠개
2026년 3월 19일 → 이천이십육년 삼월 십구일  (한자어)
5천만원         → 오천만원                  (한자어)
297%            → 이백구십칠%               (한자어)
```

## 재발 방지
- TTS 텍스트 전처리에서 단위 명사 매핑 테이블 유지
- 새 단위가 등장하면 `_NATIVE_UNITS` 리스트에 추가
- 100+ 정수는 명시적으로 한자어 fallback (실제 발음 자연스러움)
- 소수점 + 단위는 무조건 한자어 (`삼쩜육 배` 보다 `삼쩜육배` 발음이 자연스러움)
