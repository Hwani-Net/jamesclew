# P-168 — 자율 결정 정책 (OpenClaw 패턴)

## 증상
- 사소한 결정마다 AskUserQuestion으로 대표님 호출 → 작업 흐름 단절
- 대표님 지적 (2026-05-18): "결정 후보를 알려줘야 결정하지? 결정이 필요하면 외부 모델이나 서브에이전트 등을 활용해서 최대한 질문을 넣고, 결정된 방향으로 진행해. 나에게는 결재 등 꼭 내가 필요한 상황에서만 나를 호출해. 그래야 진정한 OpenClaw같은 에이전트라 할 수 있는 기반"

## 원인
- 결정 부담을 자율적으로 지지 않고 모든 변동점마다 대표님 결재 요구
- 외부 모델(Codex) + 서브에이전트(Sonnet) 합의 패턴 미활용
- AskUserQuestion이 "안전한 default 행동"이라는 오해
- 진정한 자율 에이전트의 기반 약함

## 해결 (자율 결정 패턴)

### 1. 결정 분류
- **자율 결정 (메인 단독)**: 작은 분기, 명백한 다음 단계, 우선순위 공식 적용 가능
- **외부 모델·서브에이전트 합의 결정**: 콘텐츠 품질·정합성·코드 평가 등. Codex/Sonnet에 옵션 제시 + 평가 받고 다수의견 채택
- **대표님 결재 필수**: (a) git push (b) 비용 큰 작업 (Codex 30+ 호출, 라이브 데이터 변경, 외부 결제) (c) 대표님 명시 요청 (d) 비가역 데이터 삭제 (e) 보안·시크릿 노출 위험

### 2. AskUserQuestion 사용 게이트
다음 모두 충족할 때만 호출:
- 자율 결정 시 발생 위험이 대표님 결재 시 작업 흐름 비용을 명백히 초과
- 외부 모델/서브에이전트 합의로도 결론 안 남
- 결재 후 즉시 진행 가능한 단일 분기점

### 3. 자율 결정 절차
1. 결정 후보 2-4개 정의 (우선순위 공식 적용 + 외부 모델 의견 수렴)
2. Codex 또는 Sonnet에 의견 청취 (보조 의견, 단독 결정 금지 — P-163)
3. 다수의견 또는 점수 높은 후보 채택
4. 진행 + 결과 보고
5. 대표님이 다른 방향 지시 시 즉시 정정

## 재발 방지
- 매 작업 마무리에 "다음 작업 결정 후보 N개 + 자율 채택안" 형식으로 보고
- AskUserQuestion 호출 전 "이게 정말 대표님 결재가 필요한가? 외부 모델 의견으로 충분한가?" 자가 점검
- 결재 필수 액션 5개 (위 1. 분류) 외에는 자율 진행
- post-edit/stop hook으로 "AskUserQuestion 빈도 모니터링" 검토 (남용 자동 경고)
