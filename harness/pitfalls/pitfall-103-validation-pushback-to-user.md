# 증상
처치/구현 완료 후 "대표님이 X에서 확인해주세요"로 끝맺음.
대표님 지적: "검증은 너가 호출해서 해봐야지"

# 원인
1. 외부 CLI/API 호출 경로 사전 조사 부족 → "외부 호출 불가능"을 사전에 명시 안 함
2. desktop-control 같은 GUI 자동화 옵션을 처음부터 고려 안 함
3. 처치 성공 보고를 우선시 → 검증 단계를 사용자에게 떠넘김

# 해결
처치/구현 보고 시 다음 순서 강제:
1. **검증 가능 경로 사전 식별** (CLI / API / GUI 자동화 / 로그 파싱 중 무엇으로 가능한가)
2. **즉시 검증 실행** (3회 시도 + 대안 2개 원칙)
3. **검증 결과까지 보고** — 이후 사용자 후속 액션이 필요하면 그 시점에만 안내

# 재발 방지
- "처치 완료" 메시지 작성 전, 자체 검증 시도 1회 이상 필수
- 외부 호출 불가능하면 그 사실을 처음부터 명시 (떠넘김 회피)
- ghost mode 위반 자가 점검: "확인해주세요" / "알려주세요" 단어 사용 시 알람
