# PITFALL-255 — 산출물 파일을 백틱 코드로 제시(클릭 불가), 마크다운 링크 누락

- 날짜: 2026-06-03
- 분류: 응답 포맷 / 파일 제시

## 증상
공모 기획서 등 산출물 파일 경로를 `백틱 인라인 코드`(+윈도우 절대경로 역슬래시 `D:\AI 비즈니스\...`)로 제시 → 대표님이 클릭해도 열리지 않음. "산출물로 제시한 파일링크가 제대로 보이지 않아. 클릭 시 뜨도록 수정해줘."

## 원인
시스템 규칙("When referencing files, format them as markdown links … path relative to the working directory as the href")을 어기고 **백틱 인라인 코드 + 절대 윈도우 경로(역슬래시)**로 표기. 백틱 코드는 클릭 불가, 역슬래시·절대경로 href는 링크로 인식되지 않음. 습관적으로 코드 스팬을 써서 발생.

## 해결
작업디렉터리(`D:\AI 비즈니스\공모해커톤`, bash상 `/d/AI 비즈니스/공모해커톤`) 기준 **상대경로 + 포워드슬래시**로 마크다운 링크화:
`[기획서.html](KOICA-공공데이터/기획서.html)`. 공백이 있는 상위 경로("AI 비즈니스")는 상대경로로 회피(상대 부분엔 공백 없음 → href 안전).

## 재발 방지
- 파일을 제시할 때는 **항상 마크다운 링크** `[라벨](상대경로)`. 백틱 단독 표기 금지.
- href는 **cwd 기준 포워드슬래시 상대경로**. 절대 윈도우 경로·역슬래시 금지.
- 라인 참조는 `[파일.md:42](경로/파일.md:42)` 형식.
- 이 규칙은 시스템 프롬프트에 이미 명시 — 매 응답에서 파일 언급 시 의식적으로 적용.
