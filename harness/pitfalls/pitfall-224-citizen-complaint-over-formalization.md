## 증상

- 법령 조항 5개 인용 (민법114조, 본인서명확인법6·13조, 민원처리법11조+시행령8조의2, 금소법14·16조)

- Codex 7라운드 감사 누적 토큰 약 100만+

사용자 지적: **"통상적인 민원 절차를 알기나 하고 이런 말 하는 거야? 뭘 확보하고 증거를 이거저거 제시하는 민원은 난생 처음 들어보는데?"**, **"무슨 명예훼손이니 어쩌니 민원서류에 내 감정 우선 작성하지 누가 이런 거 걱정하며 작성하냐?"**

---

## 원인

1. **통상 양식·절차 사전 검증 누락**: 민원서 작성 전에 "실제 e-금융민원센터 양식 + 시민 후기 + 입력란 분량"을 Tavily로 먼저 확인하지 않음. 양식 무지 상태에서 형식 결정.

2. **Codex 감사가 법적 정확성·입증력에만 편향**: Codex는 "법령 인용 정확성, 사실 단정 vs 인식 진술 분리, 명예훼손 우려" 같은 법률 자문서 기준으로 감사 → "이건 일반 시민 민원이지 변호사 의견서가 아니다"라는 본질적 컨텍스트 누락. 사용자가 "발송가능 판정"만 받으려고 매 라운드 권고 그대로 반영.

3. **명예훼손 과보호**: Codex가 명예훼손 위험 언급 → 본인 감정·불만 표현을 "본인이 인식한 부분", "단정할 수 없는 부분"으로 절제 → 민원의 본질(불만 호소) 표백. 실제 행정 민원은 비공개 제출 + 사실 적시 + 공익 목적으로 명예훼손 입증 매우 어려운데 과보호.

4. **시민이 할 일과 기관이 할 일 혼동**: CCTV 보존 요청, 증거 분류, 자료 첨부 등을 시민이 직접 해야 한다고 짬. 실제로는 금감원/은행이 자체 조사하고 시민은 진술만 함.

2. **e-금융민원센터 입력란에 그대로 복사·붙여넣기 가능** 구조로 전환.

## 재발 방지

### Rule 1 — 행정/민원/공공 문서 작성 시 첫 단계 강제

사용자가 "민원/진정/탄원/이의신청" 요청 시 **본문 작성 전** 반드시:

```
1. Tavily 검색: "[기관명] [민원유형] 신청 양식 작성 예시"
2. Tavily 검색: "[민원유형] 후기 시민 일반인 경험"
3. 실제 양식의 입력란 개수·분량 확인
4. 사용자에게 "통상 분량 X자, 첨부 Y개, 형식 Z"를 보고 후 형식 결정
```

### Rule 2 — Codex 감사 적용 대상 분리

Codex 법적 정확성 감사는 **변호사 의견서·소장·내용증명 등 법률 효력 직접 발생 문서**에 한정.
**일반 시민 민원서·진정서·고객 클레임**에는 다음 기준으로 대체:
- 1라운드 자가 점검: 통상 양식·분량 부합 + 사실관계 누락 없음 + 본인 감정 살아있음
- 사용자 검토 직행

### Rule 3 — 명예훼손 과보호 금지

행정기관 비공개 민원에서 명예훼손 입증 요건(공연성·허위사실·비방목적)은 매우 까다로움.
시민의 솔직한 사실 적시 + 본인 감정 표현은 자유롭게 허용.
"본인이 인식한 부분", "단정할 수 없는 부분" 같은 절제 표현은 시민 민원에 부적합.

### Rule 4 — 시민의 역할 ≠ 변호사의 역할

시민 민원에서 다음은 **시민이 할 일 아님** — 민원서에 넣지 말 것:
- 증거 자료 분류 체계 (객관/진술/확인요청 등)
- CCTV 보존 요청 절차 (필요 시 별도 행동, 민원서에 적지 않음)
- 법령 조항 인용
- 청구 사항 세분화 (1~3개 정도면 충분)

시민은: **무슨 일이 있었는지 + 왜 부당한지 + 무엇을 원하는지** 3가지만.

## 추가 사고 (2026-05-28 동일 세션, v9 작성 단계에서 발견)

v1~v8 전체에 걸쳐 다음 사실관계 검증을 사전에 하지 않음:

### 사고 내용

| 서류 | 자녀 대리발급 가능? | 사건과의 관련성 |
|------|---------------------|----------------|
| 본인서명사실확인서 | **불가능** (본인 직접만, 서초구청 공식) | v1~v8에서 "더 간단한 대안"으로 강조했으나, 본인(어머니)이 자리 비우면 자녀가 자리 지키더라도 어머니가 직접 가야 하는 현실적 제약 미점검 |
| 인감증명서 | **가능** (위임장+본인 신분증+대리인 신분증, 인감증명법 시행령 제13조) | **v1~v8 어디에서도 짚지 않음** — 사용자가 v9 단계에서 직접 지적("내가 어머니 신분증 가지고 가서 떼면 된다는 안내가 있었다면") |

### 원인

- 첫 검증 단계에서 "본인서명사실확인서 = 더 간단한 대안" 결론에 만족하고 멈춤
- 사건의 실제 제약(어머니 자리 비울 수 없음 + 자녀가 50분 왕복 부담)을 두 서류 발급 절차에 매핑해서 검토하지 않음
- Codex 감사도 법령 정확성만 봤지 "사건 현실과 매칭되는 대안인가"는 평가 항목 아님

### 추가 재발 방지 (Rule 5)

행정/금융 절차 관련 민원 작성 시:
1. 언급되는 모든 서류·절차에 대해 **"본인 vs 대리, 누가, 어디서, 무엇 들고 가야 하는가"를 표로 정리**
2. 그 표를 **사건의 실제 제약(시간·이동·동석자)과 매핑**해서 어떤 옵션이 진짜 현실적이었는지 식별
3. 그 식별 결과를 사용자에게 보고 후 민원서 작성 방향 합의

이 단계가 v1~v8에서 누락되어 사용자가 8차례 검토 후 직접 지적해야 했음.

## 추가 사고 2 (2026-05-28 동일 세션, v12 단계에서 발견 — 가장 큰 메타 사고)

### 사고 내용

v1~v11 전체에 걸쳐 **"박승일 직원이 본인서명사실확인서·인감증명서 대리발급 가능성을 안내하지 않은 안내 부실"**을 핵심 비판 사항으로 사용.

실제 사실관계 (v12 단계에서 사용자가 정정):
- **박승일 직원은 안내를 정상적으로 다 했음** (인감증명서·본인서명사실확인서·대리발급 등)
- **어머니가 신분증을 안 가져오셔서 직원이 안내한 옵션들이 모두 무산된 것**
- 즉 "안내 부실"이 아니라 **"고객 측 사정으로 옵션 사용 불가"**가 진실

11라운드 동안 잘못된 전제(직원이 안내 안 함)로 민원서 작성·감사 반복.

### 사용자 신호 누적 후 정정 시점

| 단계 | 사용자 메시지 | Claude가 해석한 방향 | 진실 |
|------|--------------|---------------------|------|
| v8 → v9 | "내가 어머니 신분증 가지고 가서 떼면 된다는 안내가 있었다면" | "자녀 대리발급 안내 누락"으로 해석 → v9에 추가 | 사용자는 "그런 안내가 있었더라면 했을 텐데"라는 가정. 실제로는 직원이 안내했고 어머니 신분증 미지참으로 무산 |
| v9 → v10 | "어머니가 신분증을 안 가져오셨네" | "신분증 미지참 사실을 본문에 추가"로 해석 → v10 작성 | 사용자는 "안내된 옵션이 신분증 때문에 무산된 정황"을 떠올린 것 |
| v10 → v11 | "신분증 관련 내용은 본문에서 제외하라" | "신분증 미지참 사실만 빼고 안내 부실 비판은 유지"로 해석 → v9 그대로 복원 | 사용자는 "신분증 관련 + 그로 인한 안내 부실 비판 = 둘 다 빼야 함"의 의미 |
| v11 → v12 | "직원은 안내를 다 해줬는데 엄마가 신분증 안 가져와서 무산됐다는 말이야. 신분증 관련 내용은 다 제거하라 했잖아" | 직접 정정 받음 → v12 작성 | 진실 |

### 원인

1. **사용자 발언의 함의를 표면적으로 해석**: "어머니가 신분증을 안 가져오셨네"라는 단서를 받았을 때, "그래서 어떤 옵션이 가능했는데 무산됐는지" 사용자에게 직접 확인하지 않음. 자체 추론으로 사실관계 재구성.

2. **이전 작업 결과물에 매몰**: v9에서 "직원 안내 부실"이 핵심 비판이었기 때문에, 그 뒤 단서들이 들어와도 그 비판 프레임 자체를 흔드는 해석은 회피.

3. **사용자가 "제외하라"고 한 것의 범위 추측**: "신분증 관련 내용 제외"가 단순히 "신분증 미지참 문장만 제외"인지 "신분증으로 인해 무산된 모든 흐름 + 그 흐름을 전제로 한 안내 부실 비판 제외"인지 명확히 확인 안 함. 가장 보수적인 해석(문장만 제외)으로 처리.

### 추가 재발 방지 (Rule 6)

사용자가 사실관계를 정정하거나 추가 정보를 줄 때:

1. **그 정보가 기존 작성된 비판/주장 프레임을 무너뜨리는지 즉시 점검**. 무너뜨리면 그 비판은 제거 대상.

2. **사용자에게 "이 사실이 들어오면 X 비판은 더 이상 유효하지 않아 보이는데, 그 비판을 제거해도 되나요?"라고 직접 확인**. 추측으로 처리하지 말 것.

3. **"제외하라"는 범위가 모호하면 정확한 범위를 물어볼 것**. 가장 보수적 해석으로 자체 결정하지 말 것.

4. **사용자가 같은 사실관계를 반복해서 언급(v8: "안내가 있었다면", v9: "신분증 안 가져오셨네", v10: "다 제거하라")할 때는, 이미 두 번째 언급 시점에서 핵심 사실관계가 흔들리고 있음을 인지하고 전체 프레임 재검토할 것**. 부분 수정 반복하지 말 것.

### 메타 교훈 보강

**도구의 감사 통과 ≠ 사실관계의 정확성.**

Codex가 v1~v11을 매번 "발송가능" 판정했지만, 그 전체가 잘못된 사실관계 위에 세워진 비판 프레임이었음. Codex는 "법령 정확성·논리 일관성·명예훼손 위험"만 평가했지, "사실관계 자체가 맞는가"는 평가 대상 아니었음.

**사실관계 확인은 사용자에게 직접 물어보는 것 외에 대체 없음**. Codex로 검증되지 않는 영역.
