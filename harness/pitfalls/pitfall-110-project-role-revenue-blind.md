# P-110: 프로젝트 새 역할 제안 시 매출 기여도 미검증 (sunk cost 사고)

## 증상
rmaszz-d0 프로젝트 자체빌드 코드를 다 폐기한 후 "이 프로젝트의 새 역할" 옵션 3개 (A 동결 / B TikTok 스크립트 생성기 / C 벤치마크 추적 도구)를 제시. 대표님 즉시 지적: "야 너의 프로젝트 새 역할은 돈을 벌 수 있는거야?"

진실: A/B/C 모두 매출에 **직접 기여 안 함**.
- A 동결 = 빈 폴더
- B = ChatGPT/Claude.ai로 무료 가능 (또 자체빌드 함정)
- C = ZIK Analytics 같은 표준 도구 존재

표준 경로에서 매출은 **스토어 + 콘텐츠 + 광고**에서 발생. 이 코드 폴더는 매출과 무관.

## 원인
1. **Sunk cost 사고** — "기존 코드 자산을 어떻게든 살려야 한다"는 매몰비용 회수 압박이 새 역할 정당화 명분을 만들어줌
2. 새 역할 제안 시 "매출에 어떻게 기여하나?" 1차 검증 누락
3. "코드 프로젝트 = 가치 있어야 함"이라는 디폴트 가정 — 실제로 코드가 매출과 무관할 수 있음을 인정 안 함
4. P-107/108/109에서 표준 경로(Shopify 앱 + 콘텐츠)에 도달했음에도, 코드 작업의 자리를 억지로 만듦

## 해결 (2026-05-04 적용)
프로젝트 새 역할 제안 전 의무 체크:
1. **매출 직결 분류**: 직접 매출 발생(앱 판매/SaaS 구독/강의 수익) / 간접 보조(콘텐츠 도구) / 무관(내부용)
2. 간접 또는 무관이면 **동결 권고가 디폴트**. 시간/리소스를 매출 활동에 쓰는 게 우선
3. 코드로 돈 벌고 싶으면 **별도 사업 모델 결정** — 기존 스토어 살리기와 분리
4. "기존 코드 활용"을 명분으로 새 역할 만들지 말 것 (sunk cost 거부)

## 재발 방지
- 코드 프로젝트 진로 결정 시 첫 질문: "이게 직접 돈을 버는가?" 답이 No면 동결이 디폴트
- 매출 직결 옵션을 제시할 때 가격/시장/경쟁자 1줄씩이라도 명시 (예: "Shopify 앱 마켓플레이스 — DSers 월 $19.90, 우리는 차별화 X")
- "프로젝트의 새 역할" 같은 메타 질문 자체를 의심 — 진짜 필요한 건 매출 활동이지 코드 작업이 아닐 수 있음

## 누적 패턴 (이번 세션)
