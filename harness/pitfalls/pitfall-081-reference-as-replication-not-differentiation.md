# P-081: 레퍼런스를 차별화 출발점으로 오해 — 복제 검증 누락

- **발견**: 2026-04-29
- **프로젝트**: 영상제작 (AI 애니메이션 트랙)
- **재발**: -

## 증상

YouTube 영상("해치의 이상한 나라" K-판타지 AI 뮤직비디오) 분석 직후 첫 벤치마크 노트(`ref-001-haechi-wonderland.md`)에 "우리 첫 작품 차별화 가설" 섹션을 작성. "K-판타지는 이미 점유됨 → 우리는 도깨비/산신령/사이버펑크 한국으로 차별화" 식 가정.

이후 대표님 교정: **"우리가 레퍼런스로 삼는 영상을 그대로 카피할 수 있어야 제대로 된 프롬프트 및 지시를 갖춘 레퍼런스가 된다. 그 후에 변형 가능. 지금은 영상을 봐서 기억하지만 시간 지나면 휘발되니, 지금 실제 영상과 동일한 레퍼런스를 복제 구현하는 게 맞다."**

## 원인

레퍼런스의 본질을 잘못 정의:
- **잘못된 정의**: 레퍼런스 = 차별화 출발점, 우리 작품의 좌표 잡는 비교 대상
- **올바른 정의**: 레퍼런스 = **복제 가능성이 검증된** 프롬프트/워크플로우 자산. 차별화는 그 위에서.

복제 검증을 건너뛰면:
1. 영상 시청 직후의 단기 기억에 의존하는 가짜 자산
2. 시간이 지나면 영상 디테일 휘발 → 자산이 비어버림
3. 차별화 가정도 기준 없이 떠다님 (무엇 대비 차별화인지 정의 불가)
4. 다른 작품에 응용·변형할 때마다 매번 처음부터 다시 추론

## 해결

새 레퍼런스 분석 시 다음 순서 강제:
1. **분석 (Phase 0)**: 파이프라인 해부, 핵심 패턴 추출
2. **복제 (Phase 1)**: 동일 스토리·캐릭터·컷·음악 톤으로 재현. 영상의 핵심 컷 N개와 우리 생성 컷 N개 나란히 놓고 매칭률 80%+ 검증
3. **자산화 (Phase 2)**: 복제 성공한 프롬프트·시트·워크플로우를 재사용 가능한 템플릿으로 정리
4. **변형/응용 (Phase 3+)**: 이때 처음으로 차별화/변주 가능

차별화 섹션을 분석 노트에 미리 작성하지 않음. Phase 3 진입 시 신규 노트로 분리.

## 재발 방지

- 새 레퍼런스 노트 작성 시 "차별화 가설" / "우리 작품 좌표" / "구별 포인트" 섹션을 Phase 0~2 단계에 포함하지 않는다
- 분석 노트 첫 줄에 **복제 미션**을 명시 (이 작품을 그대로 다시 만들 수 있는가?)
- 복제 검증 전에는 "응용·변형" 단어 사용 금지
- 영상 휘발성 기록: 지금 영상이 머릿속에 있을 때 모든 디테일을 노트로 옮긴다 (씬 단위 묘사·타이밍·컬러·소품)
