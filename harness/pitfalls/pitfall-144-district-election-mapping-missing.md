# 핵심 가치 제안 작동 불능 — districts.json 매핑 불완전

## 증상

의원다나와 (2026 지방선거 PWA)에서 사용자가 "내 주소 → 내 후보 찾기" 동선 진입 시 7개 선거 중 광역단체장 + 교육감 **2개만 표시**. 광역의원·기초의원·기초단체장·국회의원 재보궐은 매핑 누락으로 안 보임.

전수 조사 결과:
- 1,111개 동 중 1,099개(99%) 가 광역단체장 + 교육감 2개만 매핑
- 5개 이상 매핑된 동은 7개(0.6%) 뿐
- 평균 2.03개

## 원인

`build-districts.ts` 스크립트가 광역 단위 선거(metropolitan-mayor, superintendent)만 자동 매핑. 동·읍·면 단위로 분할되는 선거구(metropolitan-council, local-council, local-mayor)는 매핑 로직 부재.

선관위 공식 선거구 구역도(읍·면·동 → 선거구) 데이터 수집 안 됨. 광역의원/기초의원 선거구는 hash ID(`73b9406d` 등) 사용으로 동명으로 직접 매칭 불가.

## 해결

**1차 (D-8 빠른 fix)**: 휴리스틱 매핑 — 같은 sigungu의 모든 광역의원·기초의원·기초단체장 선거를 electionIds에 추가. 사용자가 본인 선거구를 후보 목록에서 직접 식별. /my 페이지에 "본인 선거구 선택" UI 추가 필요.

**2차 (V1.5+)**: 선관위 공식 선거구 구역도 PDF/CSV 확보 → 정확한 읍·면·동 ↔ 선거구 매핑 데이터 구축.

## 발견 경로

대표님 지적: "청주에 남일현 나가는데 왜 어디에도 없냐?" → 검색 박스에서는 발견 가능했지만, 핵심 문제는 사용자가 "내 주소 → 내 후보" 동선에서 본인 선거구 후보를 못 본다는 것. 검색은 우회로일 뿐.

## 재발 방지

- districts.json 빌드 후 평균 매핑 수 자동 체크. < 5개면 빌드 실패.
- /my 페이지 E2E 테스트: 임의 동 선택 → 광역의원/기초의원/기초단체장 후보 1명 이상 표시 검증.
