# 증상
PITFALL-122 (엔지니어링 결정 위임 금지) 직후 또 같은 실수. 32B vs 7.8B vs 사이클 간격 같은 모델/엔지니어링 트레이드오프를 외부 모델 자문 없이 옵션 ①~④로 사용자에게 던짐.

# 원인
"비용/품질 트레이드오프" 표현으로 사용자 결정 영역으로 자체 분류. 그러나 진짜 사용자 결정은 정책(완전 차단 vs 일부 허용)이고, 모델 선택/사이클 간격은 엔지니어링.

# 해결
재발: 2026-05-10 11:32 (사용자 직접 지적). PITFALL-122 본문에 명시:
> 모델/툴 선택 = 외부 모델 자문 영역

다음부터 결정 분기점에서 자체 판단:
1. 정책 결정인가? → 사용자
2. 엔지니어링 trade-off인가? → 외부 모델 자문 → 결론 모호하면 사용자
3. 검증/실행인가? → 자체

# 자체 검증
- PITFALL-122 작성 후 1시간 내 재발 — 결정 boundary 인식이 아직 굳지 않음
- 외부 모델 자문을 default workflow로 강제할 트리거 필요 (옵션 ①/②/③ 패턴 → 자문 자동 진입)
