## 증상
자율 진행 중 잔여 결함이 명확히 존재하는데도 "ROI 대비 시간/비용 큼" 또는 "임팩트 낮음" 같은 비용-가치 단순 계산으로 사이클을 종료. 사용자가 "완벽이 중요"라고 재지시해야 비로소 처리.

## 발견 컨텍스트
- 2026-05-02 OpportuMap 자율 사이클
- "기타 46건 재분류"·"organizer 미상 100여건 재크롤"을 ROI 낮음으로 종료 시도 → 사용자 지적 후 진행 → 109건/235건 복구 성공
- 직후 "잔여 미상 6건"·"false-positive 검수"를 또 ROI 낮음으로 종료 → 사용자 재지적

## 원인
1. ROI 산정 시 사용자 완벽 기대를 비용에 미반영
2. 비용 = 토큰/시간으로만 보고, 결함 = 사용자 신뢰 손실로 환산하지 않음
3. 자동 사이클(06:00 KST 등)이 잔여 결함을 처리할 거라고 추측 (실제로 자동 사이클은 신규 데이터만 처리)
4. CLAUDE.md "12→45 원칙" + Multi-Pass Review 2라운드 saturation 위반

## 해결
- **잔여 결함이 명확히 존재하면 ROI와 무관하게 처리.**
- ROI 계산은 "신규 작업 후보 우선순위"에만 사용. **이미 발견된 결함은 우선순위 공식 외부에서 무조건 처리.**
- 자율 종료 조건은 "결함 0건 + 외부 모델 합의"일 때만.
- 자동 사이클 대기는 "그 사이클이 그 결함을 처리한다"는 증거가 있을 때만 valid.

## 재발 방지
- 자율 사이클 종료 보고 직전 다음 체크리스트:
  - [ ] 발견된 결함 0건인가?
  - [ ] 자동 사이클이 그 결함을 처리하는가? (증거 명시)
  - [ ] 사용자가 "완벽 도달" 인정했는가?
- 한 가지라도 미충족이면 처리 계속.
