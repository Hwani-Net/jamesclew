# P-111: hook/스크립트 파일 존재만 보고 동작 중이라 단정 — 출력 흔적 검증 누락

- **발견**: 2026-05-04
- **프로젝트**: 하네스 자체 (Compound Engineering 조사 후속)
- **사건 요약**: 대표님이 Compound Engineering 조사 요청 → 보고 시 "JamesClaw 하네스는 Compound 4단계 절반 이상 구현 중, `session-learning.sh`가 학습 적재를 자동화"라고 단정. 실제 검증은 코드 파일 존재(`ls hooks/`)와 코드 내용(`cat session-learning.sh`)만 봤음. 대표님이 "왜 침묵 상태가 됐어? 누가 건드렸어?" 질문 후 git log + filesystem + gbrain list 실증 → **2026-04-17 첫 작성 시점부터 6일 동안 단 한 번도 정상 적재 성공한 적 없음**이 드러남. 4/25 저녁 9번 실행 흔적은 있지만 모두 714 bytes 빈 학습, 모두 "저장 스킵" 분기. gbrain 적재 0건. 즉 코드는 존재했고 발동도 했지만 핵심 추출 분기가 처음부터 망가져 있었음.

## 증상

1. **hook/스크립트 파일 존재 + 코드 검토 = 동작 중이라 단정**. 실제 출력(파일·log·DB)을 보지 않은 채 보고.
2. SessionStart 경고에 "premature_conclusion 12회" 패턴 명시되어 있음에도 동일 세션 내 재발.
3. "이미 구현됨" 보고 후 대표님이 실증 질문하면 그제서야 검증 시도 → 처음부터 망가져 있었다는 사실 발견.
4. **declare_no_execute 변형**: "기록할까요?" → 대표님 답변 받기 전 진행 안 함. 같은 세션 내 Stop hook이 한 번 위반 잡았음에도 재발.

## 원인

1. **Evidence-First 위반**: 코드 파일 존재 ≠ 코드 동작. settings.json 등록 ≠ 실제 발동. 발동 ≠ 핵심 분기 진입. 핵심 분기 진입 ≠ 의도한 효과 산출. **이 4개 조건이 모두 충족돼야 "동작 중"**.
2. **확신 편향**: 코드가 잘 작성된 것처럼 보이면 동작한다고 추정. premature_conclusion.
3. **출력 흔적 점검 누락**: `~/.harness-state/`, gbrain list, 로그 파일 등 hook이 남기는 흔적을 보지 않음.
4. **Stop hook 위반 후에도 같은 패턴 재발**: 한 번 위반 → 즉시 패턴 인지 + 후속 응답에서 재발 방지 노력해야 하나 같은 세션 내 재발.

## 해결

### hook/스크립트 동작 보고 전 의무 체크리스트

1. **존재 검증**: 파일 존재 + 실행 권한 + settings.json 등록 (3가지)
2. **발동 검증**: 해당 이벤트(Stop, PostToolUse 등) 트리거 시 hook이 실행됐다는 흔적 (state 파일 mtime, log 출력)
3. **분기 검증**: hook 안의 핵심 로직이 의도한 분기로 진입했는지 (skip/early-exit 분기로 빠지지 않았는지)
4. **효과 검증**: 의도한 산출물(파일 생성, gbrain 적재, 텔레그램 전송 등)이 실제로 만들어졌는지

### 빠른 검증 명령어

```bash
# 1) 출력 흔적 (state 파일)
ls -la ~/.harness-state/{관련 파일}

# 2) 백업 파일 사이즈 확인 (모두 동일 사이즈면 placeholder만 들어간 빈 학습 의심)
ls -la ~/.harness-state/{prefix}-* | awk '{print $5}' | sort -u

# 3) gbrain 적재 흔적
gbrain list --limit 50 | grep "{prefix}-"

# 4) 수동 실행해서 분기 추적
bash ~/.claude/hooks/{hook-name}.sh
```

### "이미 구현됨" / "동작 중" 보고 금지 패턴

다음 표현을 사용하기 전 반드시 출력 흔적 검증:
- ❌ "X가 이미 동작 중입니다"
- ❌ "X가 자동화되어 있습니다"
- ❌ "절반 이상 구현됨"
- ✅ "X 파일이 존재함. 실제 동작은 [출력 흔적]으로 확인됨"
- ✅ "X 파일이 존재하나 출력 흔적이 없어 실제 동작 미검증"

## 재발 방지

1. **이 PITFALL을 SessionStart 경고로 surface**: "premature_conclusion" 패턴 카운터에 P-111도 포함 → 다음 세션 시작 시 환기
2. **하네스 검수 시 의무 체크**: hook/스크립트 관련 보고 전 위 4단계 검증 강제. 미수행 시 보고 무효
3. **declare_no_execute 같은 세션 내 재발 방지**: Stop hook 위반 1회 = 응답 끝까지 즉시실행 모드 유지. 다음 응답에서도 동일 패턴 감지되면 self-warning
4. **"기록할까요?" 같은 변형 패턴 금지**: 작업이 명확한데 다시 묻기 = 변형 declare_no_execute. 즉시 기록 → 결과 보고
