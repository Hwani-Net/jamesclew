# P-042: /agent-team 실측 대상이 토이 수준에 갇힘

> type: pitfall | id: P-042 | date: 2026-04-18 | tags: pitfall, agent-team, scope, quality

## [P-042] /agent-team 실측이 토이 유틸리티에만 성공했다고 "영상 수준 초과" 자평
- **발견**: 2026-04-18 (v7 temp-converter 완료 후 대표님 지적)
- **증상**:
  - v2(포켓몬 계산기) → v3(색상 대비) → v4(퍼센타일) → v5(한글 자모) → v6(복리) → v7(온도 변환) 
  - 5회 연속 모두 **단일 SPA + 단일 기능 + 순수 함수** 수준
  - 디렉터(Opus)가 "영상 수준 초과 달성"이라 자평했으나 **영상 자체도 포켓몬 계산기(토이)** 였음
  - 대표님 지적: "이제야 겨우 화씨·도씨 같은걸 하기 시작했잖아. 적어도 완성형 품질 높은 무언가를 스스로 만들 수 있어야지?"
- **원인**:
  1. director가 v 검증이라는 하위 목표(규칙 확인)에만 집중해 실측 대상 복잡도를 낮게 선정
  2. "R9 수학 4단계 검증" 같은 규칙 동작 관측에 유리한 단순 프로젝트로 스코프 축소
  3. **진짜 사용자에게 가치 있는 완성형 프로젝트** (다중 화면/라우팅, 상태 관리, 디자인 시스템, SEO, 테스트 커버리지, 오프라인, 접근성) 미도전
- **해결**:
  - v8부터 실측 대상을 **본격 품질 프로젝트**로 상향:
    - 다중 화면 + 라우팅
    - localStorage/IndexedDB/Firestore 상태 관리
    - 디자인 토큰 + 컴포넌트 구조
    - 실 사용자 UX (로딩/에러/빈 상태/오프라인)
    - 테스트 커버리지 + 문서화 (README)
    - SEO + PWA
  - 첫 v8 대상: 개인 포트폴리오 사이트 또는 태스크 매니저(Kanban)
- **재발 방지**:
  - /agent-team 스킬의 "실측 대상 선정 가이드"에 복잡도 하한 명시
  - `pitfall-042` gbrain 저장으로 차후 director 판단 시 검색 가능
  - 실측 후 성과 요약에서 "영상 수준" 대신 "실사용 가치" 기준으로 평가
