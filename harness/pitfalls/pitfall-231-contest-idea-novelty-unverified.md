# PITFALL-231 — 공모전 아이디어 "독창성" 검증 없이 단정

## 증상
공모전(경북 공공데이터 창업경진대회) 아이디어 제안서에 "국내 최초 / 기존에 없던 서비스"라고
검증 없이 단정 작성. 대표님이 "Codex 교차검증했냐 / 정말 기존에 없던 거 맞냐" 지적 후 웹 검증하니
거의 동일한 서비스가 이미 다수 존재:
- 그린대로(농식품부, 귀농귀촌+농촌빈집은행, AI추천 구축예정)
- 빈집애(국토부/한국부동산원, 빈집거래+AI확산예측)
- 집잇다(정선군, "전국 최초" 빈집매칭)
- 마이세컨플레이스(클리, 빈집 공공데이터 → 충남 공공데이터 경진대회 大賞)

## 원인
- 아이디어 독창성 주장을 웹 검색 없이 작성 (premature_conclusion 패턴)
- 제안서 8섹션 + 양식 4종까지 만든 뒤에야 검증 → 매몰비용 발생
- 대회 실격규정(상용화된 과제 / 최근3년 타대회 수상작 유사) 사전 점검 누락

## 해결
1. 공모전 아이디어는 제안서 작성 **전에** 3축 웹 검증 필수:
   - (a) 기존 상용 서비스/스타트업
   - (b) 정부·공공 플랫폼 (가장 흔히 겹침)
   - (c) 최근 3년 타 지자체/부처 공공데이터 경진대회 수상작
2. Codex 교차검증으로 차별성·실격위험 객관 평가 (skip_review 금지)
3. "국내 최초/유일" 등 단정 표현은 검증 증거 없으면 사용 금지

## 재발 방지
- contest-idea 스킬 / 아이디어 제안서 작성 시 "독창성 주장 = 검증 게이트" 강제
- 독창성 섹션 작성 전 Tavily 3축 검색 결과를 근거로 첨부
- Evidence-First: "기존에 없다"는 부재 증명도 검색 증거 필요
