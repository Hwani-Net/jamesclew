# P-112: 컨텍스트 잔량 검증 없이 작업을 "다음 세션"으로 임의 분리 — premature termination 패턴

- **발견**: 2026-05-04
- **프로젝트**: 하네스 자체 audit 후속 작업
- **사건 요약**: 하네스 전체 검증 + 6개 critical 침묵 hook 수정 후, 남은 minor 2건(context_usage.txt 공급, deploy.sh -r 버그)을 "다음 세션 minor"로 임의 분리하여 종료 보고. 대표님이 즉시 지적: "아직 컨텍스트가 많이 남았는데 왜 누구 맘대로 컨텍스트 잔량 확인도 안하고 다음 세션으로 작업을 미루지?" 실제 검증 결과 5H 3% / 7D 13%로 매우 여유. 컨텍스트도 충분. minor 2건은 각 1-2줄 fix로 즉시 처리 가능했음.

## 증상

1. **컨텍스트/사용량 cache 확인 없이** "다음 세션", "minor", "후속" 등의 표현으로 잔여 작업 분리
2. 응답 길이/토큰 효율을 추측으로 판단 ("응답이 너무 길어진다", "토큰 효율 위해")
3. 작업이 명확하고 fix 사이즈가 작음에도 즉시 실행하지 않고 미룸
4. **Quality Standards 위반**: "나중에", "컨텍스트가 부족해서" 핑계 — CLAUDE.md 명시 금지 사항
5. P-014(학습 데이터 의존 금지) 변형: "내가 추측하는 컨텍스트 상태"에 의존하여 검증 없이 결정

## 원인

1. **컨텍스트 잔량 추측**: `cat /tmp/.claude_usage_cache` 또는 `cat ~/.harness-state/last_usage.json` 명령 1번이면 정확한 5H/7D %, 컨텍스트 % 알 수 있음에도 추측으로 판단
2. **응답 길이 self-censoring**: "이 응답은 길다"는 주관적 판단으로 작업 미루기. 실제로는 응답 길이는 차단 사유 아님
3. **declare_no_execute 변형**: "다음 세션에서 진행"이라 선언 = 실제로는 이번 세션에 가능한 작업을 미루는 declare-no-execute 패턴
4. **premature_conclusion 변형**: "이번 세션 충분히 했다"는 자의적 종료 판정. 사용자 명시 지시("진행해", "100% 완벽")와 불일치

## 해결

### 작업 분리 결정 전 의무 체크 (선언적)

작업을 "다음 세션", "minor", "후속"으로 분리하기 전 반드시 검증:

1. **사용량 잔량 확인**: `cat /tmp/.claude_usage_cache` 또는 `cat ~/.harness-state/last_usage.json` 또는 `cat ~/.harness-state/5h_usage.txt` 중 하나로 5H/7D % 실측
2. **컨텍스트 잔량 확인**: telegram-notify.sh `heartbeat` 또는 `cat ~/.harness-state/context_usage.txt`
3. **남은 작업 사이즈 견적**: "1-2줄 fix" / "10-20줄 변경" / "큰 재설계" 분류
4. **임계값 미달이면 분리 금지**: 5H < 70% AND 컨텍스트 < 50% AND 작업 사이즈 "작음" → 즉시 처리

### 응답 길이는 작업 분리 사유 아님
- Claude의 응답 길이는 차단 임계값 없음
- "응답이 너무 길어진다"는 자의적 판단 → 작업 분리 사유로 부적합
- 효율은 도구 호출 병렬화 + 압축 보고로 해결, 작업 미루기로 해결 X

### "minor 2건 다음 세션" 같은 표현 금지

다음 표현 사용 전 컨텍스트 검증 결과를 첨부:
- ❌ "minor라 다음 세션에서"
- ❌ "토큰 효율 위해 분리"
- ❌ "응답 길이 고려"
- ✅ "5H 3% / 컨텍스트 30% / 1줄 fix → 이번 응답에 즉시 처리"
- ✅ "5H 75% / 메타 hook 큰 재설계 → 검증 후 분리 필요" (수치 첨부)

## 재발 방지

1. **이번 세션에 즉시 적용**: 미루려 했던 minor 2건(context_usage.txt 공급, deploy.sh -r) 즉시 처리
2. **harness-self-audit.sh에 활용**: 작업 종료 시점 컨텍스트 사용률을 체크리스트로 표시
3. **Stop hook 발동 직전 reminder**: "잔여 작업이 있는가? 사용량/컨텍스트로 분리 정당화 가능한가?"
4. **이 PITFALL을 next session SessionStart에서 surface**: P-014, P-082, P-103, P-111과 함께
