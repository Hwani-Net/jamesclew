## SECTION 5: RULES

### rules/architecture.md (요약)
- Tool Priority: Built-in > Bash > MCP (비용순)
- Perplexity 4개 도구: search($0.006) / ask($0.03) / reason($0.02) / research($0.80)
- Tavily 5개 도구: search(기본값 basic+max5 강제) / crawl / extract / map / research
- NotebookLM: notebook_query(무료) / research_start / studio_create / source_add
- 전략: 빠른 팩트→Perplexity, 크롤링→Tavily, 심층+영구→NotebookLM
- Tool Budget: 50개 이하 유지 (230+에서 실패)
- Hosting: Firebase 전용
- 비용 추적: ~/.harness-state/api_cost_log.jsonl

### rules/quality.md (요약)
- 코드 변경 → 테스트 → 빌드 → 커밋 순서 강제
- 배포 후 HTTP 200 확인 필수 (hook 강제)
- 이미지: og:image CDN → Playwright → CDP 우선순위
- loading="lazy" 사용 금지 (P-001)
- Multi-Pass Review: 6개 패스 × N라운드 (최소 2라운드)
- 콘텐츠 검토 패스: 구조/SEO/독자관점/사실검증/이미지/경쟁대비
- 코드 검토 패스: 기능/보안/성능/UX접근성/사용자페인포인트
- Test Manipulation Guard: 테스트만 수정+소스 미수정 감지
- Change Tracker: 50개+ 파일 수정 시 스코프 크리프 경고
- Regression Guard: 삭제량 > 추가량 2배 + 10줄 초과 시 경고

### rules/security.md
- 소스코드 시크릿 작성 금지. 환경변수 사용.
- .env, credentials, *.pem, *.key 수정 차단 (hook)
- rm -rf, format, del /s/q → deny list

### rules/design_rubric.md
- 출처: Anthropic Harness 실험 (Opus 4.6, 2026-04)
- 4대 평가축 (각 0-10):
  1. Consistency (일관성) — CSS 토큰 체계
  2. Originality (독창성) — AI 클리셰 탈출
  3. Polish (완성도) — Typography/대비/간격
  4. Functionality (기능성) — UX 인터랙션
- 통과: 4축 모두 8점 이상
- AI 클리셰 블랙리스트: purple gradient / blur circle / 3-column feature grid 등
- scripts/evaluator.sh로 자동 평가

### P-001: loading="lazy" 헤드리스 브라우저 이미지 차단
- headless에서 스크롤 없으면 lazy 이미지 미로드

### P-002: 쿠팡 이미지 UI 오버레이 캡처
- 해결: og:image CDN URL → 800x800 직접 다운로드

### P-003: 쿠팡 제품 ID가 다른 모델을 가리킴
- 해결: Opus+Sonnet 교차 Vision 검증

### P-004: Firestore ↔ 로컬 JSON 불일치
- 해결: JSON 수정 후 createPost()로 Firestore 동기화 필수

### P-005: enforce-execution.sh 완료 보고 오탐
- 해결: 패턴을 미래형("~하겠습니다/~하겠")만 잡도록 변경

### P-006: agent-browser 기본 모드로 쿠팡 접근 불가
- 해결: Playwright launchPersistentContext + --disable-blink-features=AutomationControlled

### P-007: compact 후 세션 요약 저장하면 의미 없음
- 해결: compact 전(60-65%)에 저장 먼저 실행

### P-008: Context % 확인 방법 없다고 오보고
- 해결: user-prompt.ts에 context_pct 파일 저장 로직 추가

### P-009: 5H/7D Usage 캐시 stale 미감지
- 해결: usage 보고 시 resets_at 현재시각 비교

### P-010: MCP 끊김 시 재연결 없이 우회
- 순서: ① mcp remove+add 재연결 ② invoke 재시도 ③ curl 직접 (최후)

### P-011: crossReview 텍스트 3000자 제한으로 글 잘림
- 해결: 5000자로 확대

### P-012: 외부 모델 로테이션 규칙만 존재, 구현 없음
