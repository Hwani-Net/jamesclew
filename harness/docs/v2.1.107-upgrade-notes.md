## v2.1.101 — 하네스 핵심 변경사항

### [중요] 서브에이전트 MCP 도구 상속 버그 수정
```
Fixed subagents not inheriting MCP tools from dynamically-injected servers
```
- **의미**: `claude mcp add`로 온디맨드 추가한 서버의 도구를 서브에이전트가 이제 올바르게 상속함
- **하네스 영향**: Stitch, korean-law 등 온디맨드 MCP를 Sonnet 서브에이전트에 위임할 때 도구 미전달 문제 해결
- **권장 조치**: 온디맨드 MCP + 서브에이전트 패턴 재활성화 가능 (이전엔 에러 발생)

### [중요] Isolated Worktree 서브에이전트 파일 접근 수정
```
Fixed sub-agents running in isolated worktrees being denied Read/Edit access to files inside their own worktree
```
- **의미**: worktree 격리 서브에이전트가 자신의 worktree 파일을 읽고 편집할 수 있게 됨
- **하네스 영향**: Parallel Agent Safety (#14) — `isolation: "worktree"` 패턴이 이제 안정적으로 동작
- **권장 조치**: quality.md Parallel Agent Safety 섹션 신뢰성 향상

### [보안] POSIX `which` 명령 인젝션 취약점 수정
```
Fixed a command injection vulnerability in the POSIX `which` fallback used by LSP binary detection
```
- **하네스 영향**: 보안 강화. 별도 조치 불필요.

### [중요] `permissions.deny` 규칙 우선순위 수정
```
Fixed permissions.deny rules not overriding a PreToolUse hook's permissionDecision: "ask"
```
- **의미**: 이전에는 PreToolUse hook이 `"ask"` 반환 시 deny 규칙을 우회할 수 있었음
- **하네스 영향**: `enforce-review.sh`, `verify-deploy.sh` 등 deny 기반 hook의 보안 강화
- **권장 조치**: 기존 deny 규칙이 의도대로 적용되는지 확인 (이제 hook보다 우선)

### `/team-onboarding` 커맨드 추가
```
Added /team-onboarding command to generate a teammate ramp-up guide from local Claude Code usage
```
- **하네스 영향**: 없음 (신규 커맨드, 팀 온보딩 문서 자동 생성용)

### `/ultraplan` 클라우드 환경 자동 생성
```
/ultraplan and other remote-session features now auto-create a default cloud environment instead of requiring web setup first
```
- **의미**: `/ultraplan` 사용 전 웹에서 클라우드 환경 수동 설정 불필요
- **하네스 영향**: 고복잡도 빌드 워크플로우 진입 장벽 낮아짐 (CLAUDE.md Build Transition Rule)

### Hook 안정성 개선
```
Improved settings resilience: an unrecognized hook event name in settings.json no longer causes the entire file to be ignored
```
- **의미**: settings.json에 알 수 없는 hook 이벤트명 있어도 파일 전체가 무시되지 않음
- **하네스 영향**: hook 이름 오타 시 전체 설정 무력화 위험 제거

## v2.1.98 — 하네스 관련 변경사항

### Monitor 도구 추가
```
Added Monitor tool for streaming events from background scripts
```
- **의미**: 백그라운드 스크립트의 이벤트를 실시간 스트리밍으로 수신 가능

### Bash 권한 보안 강화 (다수)
```
Fixed a Bash tool permission bypass where a backslash-escaped flag could be auto-allowed as read-only
Fixed compound Bash commands bypassing forced permission prompts
Fixed redirects to /dev/tcp...//dev/udp... not prompting
```
- **하네스 영향**: `harness/hooks/` 내 Bash 명령 권한 검사가 더 엄격해짐
- **주의**: 기존에 auto-allow 되던 복합 명령(`&&` 조합)이 이제 프롬프트를 요청할 수 있음
